# Client handoff and delivery

A deliverable isn't done when it looks finished — it's done when the
client can use it without us in the room. The handoff is the difference
between a project that ends clean and one that generates support requests
for months. This is the pattern, abstracted across engagements.

## Package everything, not just the final file

Hand over the editable source, not only the exports: working files,
fonts (with their licenses), the color and type specs, and any linked
assets, organized so a stranger could find them. The test is simple — if
the client's next vendor opened this folder, could they keep working
without calling us? If not, it isn't packaged.

## Write the usage guide the client will actually read

A short, plain guide beats a hundred-page manual nobody opens: where the
files live, how to use the core pieces correctly, and the three or four
things *not* to do (don't stretch the logo, don't recolor the palette,
don't swap the headline font). Match the client's voice here too — the
guide is the last thing you write for them, and it should sound like the
rest of the work.

## Walk it through live, once

Do one real handoff session: screen-share, open the package, show the
guide, answer questions while you're still in context. Fifteen minutes
live prevents a week of asynchronous confusion, and it's the moment the
client's confidence transfers from "the agency has this" to "we have
this."

## Close the loop and archive

Confirm the deliverables list against the signed brief — everything
promised, delivered — and get explicit acknowledgment that the engagement
is complete. Then archive the project state and source records so the
relationship is documented and re-startable. A clean close is also the
warmest possible setup for the next engagement: the client leaves knowing
exactly what they got and that picking the work back up would be easy.

## Why this works

Most handoff pain comes from treating delivery as a file transfer instead
of a capability transfer. Packaging the sources, writing a guide in the
client's voice, walking it through once, and closing against the brief
move ownership cleanly — the work keeps living, the support tail stays
short, and the door stays open.
