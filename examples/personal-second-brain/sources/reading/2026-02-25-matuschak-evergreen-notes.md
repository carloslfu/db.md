# Notes — Matuschak, "Evergreen notes" (essay)

> Evergreen notes should be atomic — one note, one idea — so they can
> be linked and recombined without dragging unrelated context along.

> Evergreen notes should be concept-oriented, titled as a claim you
> can agree or disagree with, not as a topic folder. The title is an
> API to the idea.

> Prefer associative ontologies to hierarchical taxonomies. Don't sort
> notes into a tree; let the links be the structure.

My note: the "title as a claim" rule changed how I name notes — a
note called *"Attention residue makes context-switching expensive"*
is reusable in a way *"Focus"* never is. This essay is the bridge
between Ahrens' slip-box and how I actually run this garden in plain
files. Read it as [Andy Matuschak's working notes](https://notes.andymatuschak.org/Evergreen_notes).
